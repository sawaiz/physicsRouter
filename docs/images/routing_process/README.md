# Routing process figures

**TL;DR:** Stage-by-stage PNGs for synthetic + HALO. Regenerate: `python scripts/render_routing_process.py --halo`. Human summary: [../../BENCHMARKS.md](../../BENCHMARKS.md).

Generated by `scripts/render_routing_process.py`.

## Canonical (latest run; HALO-90 when available)

| File | Stage |
|------|--------|
| `1_placement_outline.png` | Footprints + Edge.Cuts outline polygon |
| `2_guide_topology.png` | Hypergraph topology, legal pin-access sites and capacity-negotiated section layers |
| `3_clearance_raw.png` | Atomic native detailed route plus manufacturing-gate state |
| `4_regeometry.png` | Post-connect re-geometry (spacing, multi-bend, arcs) |
| `5_by_layer.png` | Same copper split by layer |
| `6_process_strip.png` | Four-panel montage of stages 1–4 |
| `7_drc_map.png` | Exact native DRC (shorts / spacing / outline) on committed copper |
| `drc_report.json` | Machine-readable HALO-90 route + exact native DRC |
| `halo90_route.json` | Serialized pre-v2 HALO regression artifact, including PathFinder iteration metrics and the no-via-in-pad checkpoint; it is 12/23 and not a completed autoroute |

Tagged copies (`synthetic_*`, `halo90_*`) keep both board sets from the last dual run.

## Regenerate

```bash
# From repo root (venv + native C++ module built)
bash scripts/build_native.sh
PYTHONPATH=native/build:src python scripts/render_routing_process.py --halo
```

Metadata: `render_meta.json` (both boards, timings, route status, DRC),
`drc_report.json` (canonical HALO-90 report), and `halo90_route.json` (the
matching route object used for external KiCad validation).

The renderer writes stage 7 and `drc_report.json` directly. Copper-area fill
and thermals are deferred to KiCad; the native report validates area
boundaries plus committed tracks and vias.
